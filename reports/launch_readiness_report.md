# BuzzCalculus 目前進度與上線評估報告

日期：2026-06-05

## 一句話結論

BuzzCalculus 已經接近「本機 / 小範圍試用版」完成，可以拿來實際練習；但如果目標是公開上線給多人長期使用，仍需要補齊部署、穩定性測試、資料保護、視覺回歸與基本錯誤監控。

目前判斷：

- 本機可用 MVP：85% 左右
- 小範圍 beta：70% 左右
- 正式公開上線：55% 到 65%

## 離上線還差什麼

### A. 必須補的上線前項目

1. 實機瀏覽器 QA
   - 桌機 Chrome / Edge
   - 手機 Chrome / Safari
   - 橫向 / 直向
   - 黑板書寫、全螢幕、選擇題、LaTeX 預覽都要走一次

2. 視覺穩定性
   - 首頁剛簡化過，需要重新確認沒有版面斷裂
   - 選項 LaTeX 需用多種長答案測試
   - 手機黑板底部答案欄要確認不被鍵盤遮住

3. 部署方式
   - 決定 GitHub Pages、Netlify、Vercel 或本機伺服
   - 確認路徑、CDN、UTF-8、快取刷新
   - 建立正式 URL

4. CDN 依賴
   - 目前 KaTeX / lucide 走 CDN
   - 正式上線建議改成本機 vendor 檔或加上完整版本鎖定與 fallback

5. 測試補強
   - 目前有題庫驗證和首頁 smoke test
   - 還缺答題流程 smoke test
   - 還缺答案檢查器測試
   - 還缺選擇題選項產生測試

### B. 公開上線才需要的項目

1. 後端 / 帳號系統
   - 目前所有資料都只存在本機
   - 換裝置需要手動匯出 / 匯入
   - 若要排行榜、跨裝置紀錄、防作弊，需要後端

2. 防作弊
   - 現在只有前端切分頁偵測
   - 前端規則只能做練習約束，不能做正式競賽防作弊

3. 錯誤監控
   - 目前沒有線上 error reporting
   - 公開上線建議至少加入簡單錯誤收集或 console error 回報方式

4. 題庫品質審核
   - 題目格式已驗證
   - 仍需要人工抽查答案、解法、難度、tag 是否合理

## 建議上線路線

### Phase 1：本機穩定版

目標：自己和少數人可穩定使用。

待做：

- 手動完整跑一局：手寫、選擇題、每日、錯題
- 手機實測黑板與選項
- 修掉首頁和作答頁明顯 UI 問題
- 抽查 30 到 50 題答案

預估距離：1 到 2 天

### Phase 2：公開 beta

目標：可以丟給同學或朋友試用。

待做：

- 部署到正式 URL
- 加入版本號 / 更新日期
- 加入簡單使用說明或 FAQ
- 加入測試 checklist
- 固定 KaTeX / icon 資源策略

預估距離：3 到 5 天

### Phase 3：正式產品版

目標：可長期公開使用。

待做：

- 後端帳號與雲端紀錄
- 排行榜或班級管理
- 更完整測試
- 錯誤監控
- 題庫審核流程
- 管理題庫工具

預估距離：2 到 4 週，取決於是否要做後端與排行榜

## 建議下一步

最務實的下一步不是再加功能，而是做一輪上線前收斂：

1. 先固定首頁與答題頁 UI，不再大改 layout
2. 做一份手動 QA checklist
3. 用手機完整跑 4 種流程：手寫、選擇題、每日、錯題
4. 抽查高階題庫答案
5. 選一個部署方式，先上 beta URL

## 總結

目前 BuzzCalculus 已經不是單純原型，核心功能、題庫、紀錄、手寫、選擇題、作業本都已經具備。距離「能用」很近，距離「正式公開上線」主要差在穩定性驗證、部署、資料安全與後端能力。

如果目標是先給少數人試用，現在已經接近可上 beta；如果目標是正式競賽或多人長期使用，還需要補後端與測試基礎。
